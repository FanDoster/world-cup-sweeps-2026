# Favicon Set Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate a full multi-size favicon set from a face photo and wire it into index.html, replacing the current inline SVG soccer-ball icon.

**Architecture:** Copy the source image into the repo immediately (it lives in a temp dir), run `rembg` to strip the white background, then use ImageMagick 7 (`magick`) to produce all PNG sizes plus a multi-layer ICO. No build step — all outputs are static files committed to the repo.

**Tech Stack:** `rembg` (AI background removal, Python CLI), ImageMagick 7 (`magick` command, already installed at `/opt/homebrew/bin/magick`), plain HTML for the index.html wiring.

## Global Constraints

- All output files go in `favicon/` at the project root
- ImageMagick 7 command is `magick`, not `convert`
- No build step — every generated file is committed as a static asset
- `apple-touch-icon.png` must have a solid `#070b10` background (iOS renders transparent icons poorly)
- All other PNGs keep transparency
- `.surgeignore` already allows `favicon/` through — do not modify it

### Task 3: Generate all PNG sizes and favicon.ico

**Files:**
- Create: `favicon/favicon-16x16.png`
- Create: `favicon/favicon-32x32.png`
- Create: `favicon/favicon-48x48.png` (intermediate for ICO, also kept)
- Create: `favicon/apple-touch-icon.png`
- Create: `favicon/android-chrome-192x192.png`
- Create: `favicon/android-chrome-512x512.png`
- Create: `favicon/favicon.ico`

**Interfaces:**
- Consumes: `favicon/source-transparent.png`
- Produces: all favicon PNG/ICO files

- [ ] **Step 1: Generate small browser PNGs (16, 32, 48)**

```bash
magick favicon/source-transparent.png -resize 16x16 -background none -gravity center -extent 16x16 favicon/favicon-16x16.png
magick favicon/source-transparent.png -resize 32x32 -background none -gravity center -extent 32x32 favicon/favicon-32x32.png
magick favicon/source-transparent.png -resize 48x48 -background none -gravity center -extent 48x48 favicon/favicon-48x48.png
```

- [ ] **Step 2: Generate apple-touch-icon (180×180, dark background)**

```bash
magick favicon/source-transparent.png -resize 160x160 -background "#070b10" -gravity center -extent 180x180 -flatten favicon/apple-touch-icon.png
```

Note: resizes the face to 160×160 then centers on a 180×180 dark canvas with 10px padding on each side — prevents the face being clipped at rounded corners on iOS.

- [ ] **Step 3: Generate Android/PWA PNGs (192, 512)**

```bash
magick favicon/source-transparent.png -resize 192x192 -background none -gravity center -extent 192x192 favicon/android-chrome-192x192.png
magick favicon/source-transparent.png -resize 512x512 -background none -gravity center -extent 512x512 favicon/android-chrome-512x512.png
```

- [ ] **Step 4: Build multi-layer favicon.ico**

```bash
magick favicon/favicon-16x16.png favicon/favicon-32x32.png favicon/favicon-48x48.png favicon/favicon.ico
```

- [ ] **Step 5: Verify all files exist and have correct dimensions**

```bash
for f in favicon/favicon-16x16.png favicon/favicon-32x32.png favicon/favicon-48x48.png favicon/apple-touch-icon.png favicon/android-chrome-192x192.png favicon/android-chrome-512x512.png favicon/favicon.ico; do
  echo -n "$f: "; magick identify "$f" | awk '{print $3}'
done
```

Expected output:
```
favicon/favicon-16x16.png: 16x16
favicon/favicon-32x32.png: 32x32
favicon/favicon-48x48.png: 48x48
favicon/apple-touch-icon.png: 180x180
favicon/android-chrome-192x192.png: 192x192
favicon/android-chrome-512x512.png: 512x512
favicon/favicon.ico: 16x16 (first layer; ICO contains 3 layers)
```

- [ ] **Step 6: Commit**


### Task 4: Write site.webmanifest and wire index.html

**Files:**
- Create: `favicon/site.webmanifest`
- Modify: `index.html` line 7 (the `<link rel="icon" href="data:image/svg+xml,...">` line)

**Interfaces:**
- Consumes: PNG files from Task 3
- Produces: complete favicon wiring in the HTML

- [ ] **Step 1: Write site.webmanifest**

Create `favicon/site.webmanifest` with this exact content:

```json
{
  "name": "World Cup 2026 Sweepstakes",
  "short_name": "WC2026",
  "icons": [
    { "src": "/favicon/android-chrome-192x192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/favicon/android-chrome-512x512.png", "sizes": "512x512", "type": "image/png" }
  ],
  "theme_color": "#070b10",
  "background_color": "#070b10",
  "display": "standalone"
}
```

- [ ] **Step 2: Update index.html**

In `index.html`, replace the single line:
```html
  <link rel="icon" href="data:image/svg+xml,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 64 64'>...(long SVG string)...">
```

With:
```html
  <link rel="icon" type="image/x-icon" href="favicon/favicon.ico">
  <link rel="icon" type="image/png" sizes="32x32" href="favicon/favicon-32x32.png">
  <link rel="icon" type="image/png" sizes="16x16" href="favicon/favicon-16x16.png">
  <link rel="apple-touch-icon" sizes="180x180" href="favicon/apple-touch-icon.png">
  <link rel="manifest" href="favicon/site.webmanifest">
  <meta name="theme-color" content="#070b10">
```

The SVG line is line 7 in `index.html` — it starts with `  <link rel="icon" href="data:image/svg+xml,`.

- [ ] **Step 3: Open index.html in a browser and verify**

Open `index.html` directly in Chrome/Safari. Check:
- Browser tab shows the face photo favicon (not the old soccer ball)
- No console errors about missing favicon files

Note: `file://` paths work for local testing since there's no build step.

- [ ] **Step 4: Commit**

```bash
git add favicon/site.webmanifest index.html
git commit -m "feat: wire full favicon set into index.html"
```
